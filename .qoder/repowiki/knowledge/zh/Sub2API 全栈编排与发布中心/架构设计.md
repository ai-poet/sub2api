- **统一构建编排**：根目录 `Makefile` 作为顶层入口，通过委托调用子模块（backend, frontend）的本地构建脚本，实现跨语言技术栈的统一编译。
- **单体容器化封装**：根 `Dockerfile` 采用多阶段构建策略，并行编译前端静态资源与 Go 后端二进制文件，并将前端产物嵌入后端镜像，同时集成支付服务（sub2apipay）的运行时环境，实现单容器部署全栈能力。
- **自动化发布治理**：通过 `.goreleaser.yaml` 定义跨平台二进制打包与多架构 Docker 镜像（amd64/arm64）推送逻辑，配合 GitHub Actions 实现从版本打标、安全扫描到多仓库（DockerHub/GHCR）分发的端到端自动化。
