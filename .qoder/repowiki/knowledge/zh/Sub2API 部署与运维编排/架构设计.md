- 采用多阶段 Dockerfile (`Dockerfile`) 构建，分离前端 Node.js 构建与后端 Go 编译，最终产出基于 Alpine 的最小化镜像。
- 通过 `docker-compose.yml` 及其变体（`local`, `standalone`）定义应用、PostgreSQL 和 Redis 的服务拓扑与健康检查依赖。
- 提供自动化 Shell 脚本体系：`docker-deploy.sh` 负责生成密钥并拉取配置，`install.sh` 处理 Linux 环境下的二进制分发、用户创建及 systemd 服务注册。
- 入口点脚本 `docker-entrypoint.sh` 实现了权限降级（root 到 sub2api）、数据库迁移自动执行以及主进程与支付子进程的协同管理。
